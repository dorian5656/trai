# AI 视频生成模型参数说明

## 1. 核心参数术语

### Diffusers
**Diffusers** 是 Hugging Face 推出的一个 Python 库，用于加载和运行扩散模型（Diffusion Models）。
- **作用**: 它是目前最流行的 AI 绘画/视频生成框架，支持 Stable Diffusion, Flux, Wan2.1 等多种模型。
- **优势**: 提供了统一的 API，方便开发者快速集成和切换不同的模型，同时内置了多种调度器（Schedulers）和优化技术。

### BF16 (Brain Floating Point 16)
**BF16** 是一种 16 位浮点数格式（bfloat16），由 Google Brain 开发。
- **特点**: 它保留了 32 位浮点数（FP32）的指数位（8位），但截断了尾数位。这意味着它的**数值范围**与 FP32 相同，但精度较低。
- **用途**: 在深度学习训练和推理中，BF16 比传统的 FP16（半精度浮点数）更不容易溢出（NaN），且显存占用仅为 FP32 的一半，计算速度更快。
- **Trai 系统**: 我们在 Wan2.1 模型中默认启用 BF16，以在保持生成质量的同时提升速度并降低显存需求。

### S2V (Speech to Video)
**S2V** 是 **Speech-to-Video**（语音生视频）的缩写。
- **定义**: 一种多模态生成技术，根据输入的语音音频（和可选的参考图像/人像），生成与之口型同步、表情自然的说话视频。
- **应用**: 数字人播报、虚拟主播、角色动画等。
- **流程**: 音频特征提取 -> 驱动人脸模型 -> 生成视频帧。

---

## 2. 常见任务简写

| 缩写 | 全称 | 中文含义 | 示例 |
| :--- | :--- | :--- | :--- |
| **T2V** | Text-to-Video | 文生视频 | 输入 "一只猫在跳舞" -> 输出视频 |
| **I2V** | Image-to-Video | 图生视频 | 输入一张猫的照片 -> 输出猫动起来的视频 |
| **TI2V** | Text-Image-to-Video | 图文生视频 | 输入照片 + "猫变成老虎" -> 输出变换视频 |
| **T2I** | Text-to-Image | 文生图 | 输入 "风景" -> 输出图片 (如 SD, Midjourney) |

## 3. Wan2.1 模型参数

- **Frame Num**: 生成的帧数 (如 81 帧)。
- **Steps**: 采样步数。步数越多画质越好，但耗时越长 (推荐 20-30)。
- **Guide Scale**: 提示词引导系数。数值越大，视频越贴合提示词，但过大会导致画面僵硬 (推荐 5.0)。
