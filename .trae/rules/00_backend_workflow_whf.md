# TRAI 项目工作流与 Git 规范
> **后端规则**: 本文件属于python3规范体系, 适用于所有后端开发人员.
## 1. 提交规范 
- **类型**: `新功能`, `修补`, `文档`, `格式`, `重构`, `构建`.
- **原则**: 
  1. **按文件夹提交**: 不同模块(文件夹)必须分开提交, 禁止一次性提交所有更改.
  2. **中文日志**: 提交信息 (`subject`) 必须使用中文描述.
  3. **流程**: 先更新**各模块目录下**的 `README.md` 日志 (后端/前端/客户端) -> 再同步更新**根目录** `README.md` (聚合所有日志) -> 再用中文写提交日志 `git commit` -> 最后**手动执行** `git push`.
  4. **标点**: 更新 `README.md` 日志时, 必须统一使用**英文标点符号** (如 `.`, `,`, `:`).
  5. **小标题**: 文档中的关键步骤引导语 (如 `#### 安装步骤`) 必须使用**四级标题**, 以便在大纲中显示.
- **示例**: `新功能(auth): 添加jwt支持`

## 2. 文档更新
- **规则**: 提交必须更新 `README.md` 日志.
  1. **模块优先**: 必须先在 `backend/README.md`, `frontend/README.md` 或 `backend/pyqt_app/README.md` 中记录日志.
  2. **根目录聚合**: 根目录 `README.md` 的日志必须从各模块 README 中同步, 并在时间戳后添加后缀 (如 `_后端`, `_前端`, `_客户端`).
  3. **时间戳**: 必须使用三级标题 `### YYYY_MM_DD_HHMM` 格式 (**倒序排列**).
  4. **严禁预估**: 必须使用命令获取系统当前精准时间.
- **分类**: 日志内容必须标注分类: `后端-内容`, `前端-内容` 或 `数据库-内容`.
- **命令**: `python -c "import datetime; print(datetime.datetime.now().strftime('%Y_%m_%d_%H%M'))"`
- **示例**: 
  ```markdown
  ### 2026_01_26_1555_后端
  - **后端**: 更新API规范.
  ```
