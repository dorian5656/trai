# TRAI 工作流与 Git 规范
> **后端规则**: 适用于所有后端开发人员.

## 1. 提交规范
- **类型**: `新功能`, `修补`, `文档`, `格式`, `重构`, `构建`.
- **原则**: 
  1. **按模块提交**: 禁止一次性提交所有更改.
  2. **中文日志**: `subject` 必须中文.
  3. **流程**: 更新模块文档 -> 聚合根文档 -> `git commit` -> 手动 `git push`.
- **示例**: `新功能(auth): 添加jwt支持`

## 2. 核心索引
- **文档规范**: 详见 [07_Doc](./07_backend_doc_whf.md).
- **分支**: `main`(稳), `dev`(开), `feature/*`(功).
